uniapp push

1. Configuration push

Open the project's manifest.json file

In the App SDK configuration module, check unipush push

Check push in the APP module permission configuration

Enter the configuration page

Please confirm real-name authentication during configuration

2. Configuration Information

1. Get the Android signature

(1) https://ask.dcloud.net.cn/article/35777
(2) Win10 signature needs to be written into a file, which requires administrator privileges

Generate a signature named test

In this case, just copy the command given by the system and execute it.
Note that administrator privileges are required here.

After completion, two keystore files will be generated, of which the old suffix is ​​a backup file

You can use the following command to view:
keytool -list -v -keystore test.keystore
Enter keystore password: //Enter the password and press Enter
to output the following format:

Among them, SHA1 and SHA256 are required

2. Configure push information

1. The package name needs to be remembered, find the application that needs to be configured, and configure the previously generated SHA1 to the application signature

3. Package Execution

The package name and alias must be consistent in the front

Waiting for packaging to complete


Change the packaging base when done

Run to device

3. Test push

Send a push in the official website

Please note that notification permissions need to be enabled

4. Configure offline vendor push

1. Register a mobile phone manufacturer account (Huawei)

Huawei Developer Platform Account Registration: (It is recommended to use a Huawei account directly, otherwise push may not be available)
1. Go directly to https://developer.huawei.com/Huawei Developer Alliance to register an account and follow the prompts step by step

2. After registering an account, go to the Management Center ----》My Applications ----》New to create a new project.

3. After the new creation is completed, click Development to enter the following page

4. Fill in the package name. This package name is very important and needs to be consistent with the unipush package name .

5. After the project is created, you need to generate a fingerprint certificate file and fill in the previously generated SHA256 into the marked part as shown in the figure

After the project information is completed, the push service is enabled .

Click "Activate Now"

web push proxy open

2. Register a mobile phone manufacturer account (Xiaomi)

1. Register an account on the Xiaomi development platform: A Xiaomi account is required, and non-Xiaomi accounts will not have permissions

2. After completing the account registration, you can go to the push operation platform. Click Create Application to create your own push project. Please note that the registration must be consistent with unipush.

3. Click on the application information to get it and push the required application information

3. Unipush access manufacturers:

1. Find the previously configured unipush

2. Configure the manufacturer channel: Click the manufacturer push setting to fill in the application information of each manufacturer in the corresponding item and save.
